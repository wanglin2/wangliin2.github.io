# MediasMobile 媒体组件选择预览组件

依赖于vant

适用于浏览器和APP

支持类型：img（图片）、video（视频）、audio（音频），录音只支持手机。

原生拍摄的视频上传需要使用特定的上传方法

需要下载的视频预览需要使用安卓的方法来下载完后再调安卓的插件播放

安卓基于hatom，http://hatom.hikyun.com/#/index，创建APP时模板选择：定制模板-ifar构架店外空白模板v2.x

# 安装

$ npm i medias-mobile

# 引入

import MediasMobile from 'medias-mobile'
Vue.use(MediasMobile)

# 基础用法

基础用法
<template>
  <div class="container">
    <div class="content">
        <MediasMobile
            ref="MediasMobile"
            v-model="testValue"
            :mode="'select'"
            :enableVideo="true"
            :enableRecording="true"
            :base64="true"
            :max="6"
            :audioMax="3"
            :size="1"
            :audioLoop="true"
        ></MediasMobile>
    </div>
  </div>
</template>

<script>
export default {
  data () {
    return {
      testValue: []
    }
  }
}
</script>

# API

# 属性

参数说明类型可选值默认值
mode模式Stringselect（选择预览模式）、show（只读显示模式）select
valueshow模式下要显示初始的数据，数组类型，每一项为对象类型，对象字段格式详见表1-1Array—[]
fileListselect选择预览模式下要回显的数据，也是v-model绑定的数据，如果是新选择的文件，会带上一个waitUpload=true的属性，可根据该字段来筛选新选择的文件，数组类型，每一项为对象类型，对象字段格式详见表1-2Array—[]
enableVideo允许选择和拍摄视频Boolean—false
enableRecording允许录音Boolean—false
max图片和视频数量限制Number—
audioMax录音数量限制Number—
size图片文件大小限制，单位：M，只在web端有效Number—
videoSize视频文件大小限制，单位：M，只在web端有效Number—
correct是否开启图片角度矫正，开启此配置后返回的文件类型会变成base64格式Boolean—false
compressWidth开启图片压缩，该属性设置图片压缩后的宽度，开启此配置后返回的文件类型会变成base64格式Number—
compressHeight开启图片压缩，该属性设置图片压缩后的高度，开启此配置后返回的文件类型会变成base64格式Number—
base64是否以base64格式输出Boolean—false
column图片显示列数Number1-53
marginRight图片显示间距，单位：pxNumber—10
audioLoop音频是否循环播放Boolean—false
downloadVideoAutoPlay需要下载的视频下载完成后是否自动播放Boolean—false
audioRecordTime录音时长，单位：sNumber—60
videoRecordTime拍摄视频时长，单位：sNumber—15
