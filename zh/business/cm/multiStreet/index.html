# 【CM】MultiStreet 多街道

页面右上角会出现一个多街道的下拉选择列表，第一次进入页面默认会选中第一个街道，后续选择的街道数据存储在localstorage里，再次进入会选中上次选中的街道，街道变化后会触发事件，页面监听该事件来刷新本页面的数据。

# 安装

$ npm i multi-street

# 引入

import MultiStreet from 'multi-street'
Vue.use(MultiStreet)

# 基础用法

1.首先要求给vue的原型上挂载一个用于事件订阅通知的实例：Vue.prototype.$bus = new Vue()。这一步通常是在main.js里进行。

2.进入页面前需要先获取一下街道数据及是否存在上一次选择过的街道。

具体获取街道数据的接口每个组件都不一样，所以需要自己实现，街道数据需要存储到vuex里：

export const SET_STREET_INFO = 'SET_STREET_INFO'

{
    state: {
        streetInfo: null
    },
    mutations: {
        [SET_STREET_INFO] (state, streetInfo) {
            state.streetInfo = streetInfo
        }
    },
    actions: {
        async setStreet (context) {
            let { data } = await api.fetchStreet()
            context.commit(SET_STREET_INFO, data)
            return data
        }
    }
}

街道数据字段格式如下：

[
    {
        'indexCode': '04607dafca774e768f5a0f9cb708b22f',
        'name': '小河街道'
    }
]

然后需要在页面进入前获取上一次选择的街道数据，通常做法如下：

1）.在initApp.js的initApp方法里调用组件工具方法getSetStreetInfo('存储key')获取上次选择的街道数据，如果数据存在什么也不做

2）.如果数据不存在，那么就将获取到的街道数据，调用组件工具方法setSetStreetInfo进行存储，默认是存储第一项数据

示例代码如下：

import { setSetStreetInfo, getSetStreetInfo } from 'multi-street/src/utils'

async function initApp (Vue) {
    // ...

    let streetInfo = await store.dispatch('setStreet')
    if (!getSetStreetInfo('ocos')) {
        setSetStreetInfo(streetInfo[0], 'ocos')
    }

    // ...
}

3.在需要显示多街道切换的页面上引入组件。

一般都是在h-page-header组件里面引入：

<template>
    <h-page-header slot="pageHeader" :breadcrumb="i18nBreadcrumb">
        <template slot="pageHeaderAction">
            <MultiStreet prefix="ocos"></MultiStreet>
        </template>
    </h-page-header>
</template>

如果用户切换了街道后会触发MULTI_STREET_CHANGE事件。你可以监听该事件，然后刷新数据。

this.$bus.$on('MULTI_STREET_CHANGE', this.refresh)。

也可以直接在组件上绑定事件：

<template>
    <h-page-header slot="pageHeader" :breadcrumb="i18nBreadcrumb">
        <template slot="pageHeaderAction">
            <MultiStreet prefix="ocos" @change="refresh"></MultiStreet>
        </template>
    </h-page-header>
</template>

如果在页面里需要获取当前选择的街道数据，使用上述提到的getSetStreetInfo方法即可。

4.请求带上当前街道id

可使用工具方法requestAddStreetId来添加，在httpInstance.js文件的请求拦截器里添加：

// 请求拦截器
http.interceptors.request.use(function (config) {
  config = requestAddStreetId(config, 'streetIndexCode', 'ocos')
  return trimOnlySpace(config)
}, function (error) {
  return Promise.reject(error)
})

# API

# 属性

参数说明类型可选值默认值
prefixlocalstorage存储的key名称前缀，需要保持唯一性，不要和其他组件冲突—

# 事件

事件说明回调函数
MULTI_STREET_CHANGE切换街道切换到的街道数据

# 工具方法

import { setSetStreetInfo, getSetStreetInfo, requestAddStreetId } from 'multi-street/src/utils'
方法名说明参数
setSetStreetInfo存储用户选择的街道数据data：用户选择的街道数据、uid：localstorage存储的key名称前缀
getSetStreetInfo获取用户上一次选择的街道数据uid：localstorage存储的key名称前缀
requestAddStreetId请求添加多街道id入参config：axios的请求拦截器返回的config数据、field：后端约定的街道id入参，默认为'streetIndexCode'、uid：localstorage存储的key名称前缀
