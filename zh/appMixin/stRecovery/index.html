# stRecovery 滚动位置恢复

使用前端路由，当切换到新路由再回来时，一般会想要保持原先的滚动位置，否则用户体验很不好，vue-router有提供相关功能，但是一直没怎么理解，而且时有效时无效，滚动恢复说白了就是离开时记住离开前的位置，回来时再跳到记录的位置就好了。

mixin代码如下：

const mixin = {
    activated () {
        this.recoveryScrollTop();
    },
    beforeRouteLeave (to, from, next) {
        this.cacheScrollTop();
        next();
    },
    methods: {
        // 恢复
        recoveryScrollTop () {
            try {
                if (this.cacheScrollTopName && this.$store.state.common_data[this.cacheScrollTopName]) {
                    this.$store.state.common_data[this.cacheScrollTopName].forEach((item) => {
                        item.el.scrollTop = item.st;
                    });
                }
            } catch (error) {
                console.log(error);
            }
        },

        // 缓存
        cacheScrollTop () {
            try {
                if (this.cacheScrollTopName && this.cacheScrollTopList) {
                    this.$store.commit('setCommonData', {
                        [this.cacheScrollTopName]: this.cacheScrollTopList.map((item) => {
                            return {
                                el: item,
                                st: item.scrollTop
                            };
                        })
                    });
                }
            } catch (error) {
                console.log(error);
            }
        }
    }
};

export default mixin;

数据存储到vuex，vuex配置详见：/zh/fragment/data/#%E5%AD%98%E5%82%A8%E6%95%B0%E6%8D%AE%E5%88%B0store。

你的组件需要实现以下：

{
    mixins: [stRecoveryMixin],
    data() {
        return {
            cacheScrollTopName: 'STAFF_LIST_SCROLL_TOP',// 数据存储的key
            cacheScrollTopList: []// 需要缓存滚动位置的dom元素列表
        }
    }
}
