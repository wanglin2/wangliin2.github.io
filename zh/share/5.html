# lerna入门

# 主页

https://github.com/lerna/lerna

# 简述

当你开发一个较复杂的模块的时候，往往会进行拆分，拆分成更小的模块分散到各个仓库下，这样的坏处是当模块多了之后很难管理，比如修改了一个模块，另外依赖该模块的模块需要手动去升级发布，而且具体有多少依赖该模块的模块也很难理清，lerna就是解决这个问题的一个工具

它规定了特定的项目结构，主要来说就是把所有模块都放在一个仓库下进行管理，通过lerna提供的命令来进行模块依赖、安装、更新、发布

# 安装

npm i -g lerna

# 常用命令

1.为所有模块安装依赖

lerna bootstrap

2.发布

lerna publish # 用于发布更新
lerna publish --skip-git # 不会创建git commit或tag
lerna publish --skip-npm # 不会把包publish到npm上

3.初始化项目

lerna init # 固定模式(Fixed mode)默认为固定模式，packages下的所有包共用一个版本号(version)
lerna init --independent # 独立模式(Independent mode)，每一个包有一个独立的版本号

4.为packages文件夹下的package安装依赖

lerna add <package>[@version] [--dev] # 命令签名
# 例如
lerna add module-1 --scope=module-2 # 将 module-1 安装到 module-2
lerna add module-1 --scope=module-2 --dev # 将 module-1 安装到 module-2 的 devDependencies 下
lerna add module-1 # 将 module-1 安装到除 module-1 以外的所有模块
lerna add babel-core # 将 babel-core 安装到所有模块

# 基本工作流

1.创建项目文件夹

mkdir lerna-demo

2.进入文件夹并进行初始化

cd lerna-demo
lerna init

生成的目录结构如下：

- packages(目录)
- lerna.json(配置文件)
- package.json(工程描述文件)

3.在packages目录下创建各个package，如：

.
├── lerna.json
├── package.json
└── packages    
	├── house    
	│   ├── index.js    
	│   ├── node_modules    
	│   └── package.json    
	└── window        
		├── index.js        
		├── node_modules        
		└── package.json

4.添加依赖

lerna add lodash --scope=house//只给house添加依赖lodash

lerna add window --scope=house//将window添加到house的依赖里

lerna会自动检测到window隶属于当前项目，直接采用symlink的方式关联过去

5.发布到npm

lerna publish

6.更新模块

假如只修改了window模块

→ lerna updated
lerna info version 2.9.1
lerna info Checking for updated packages...
lerna info Comparing with v1.0.9.
lerna info Checking for prereleased packages...
lerna info result
- jx-house
- jx-window

lerna会自动检测哪些组件发生了改变
