# Preset预设

# 定义

一个包含创建新项目所需预定义选项的json对象

默认preset放在home目录下的配置文件：~/.vuerc

# 示例

{
  "useConfigFiles": true,
  "cssPreprocessor": "sass",
  "plugins": {
    "@vue/cli-plugin-babel": {},
    "@vue/cli-plugin-eslint": {
      "config": "airbnb",
      "lintOn": ["save", "commit"]
    },
    "@vue/cli-plugin-router": {},
    "@vue/cli-plugin-vuex": {}
  },
  "configs": {
     "vue": {},
     "postcss": {},
     "eslintConfig": {},
     "jest": {}
  }
}

当useConfigFiles为true，configs的值会被合并到vue.config.js中

插件管理

  1. 显式地指定用到的插件的版本

    {
  "plugins": {
    "@vue/cli-plugin-eslint": {
      "version": "^3.0.0",
      // ... 该插件的其它选项
    }
  }
}

  2. 允许插件的命令提示

    {
  "plugins": {
    "@vue/cli-plugin-eslint": {
      // 让用户选取他们自己的 ESLint config
      "prompts": true
    }
  }
}

# 远程preset

可以发布一个preset到git repo，仓库应该包含：

  • preset.json：包含preset数据的主要文件（必须）
  • generator.js：一个可以注入或修改项目中文件的generator
  • prompts.js：一个可以通过命令行对话为generator收集选项的prompts文件

使用远程preset

# 从 GitHub repo 使用 preset
vue create --preset username/repo my-project

# GitLab 和 BitBucket 也是支持的。如果要从私有 repo 获取，请确保使用 --clone 选项
vue create --preset gitlab:username/repo --clone my-project
vue create --preset bitbucket:username/repo --clone my-project

加载文件系统中的preset

如果--preset选项的值是一个相对或绝对文件路径，或是以.json结尾，则会加载本地preset

# ./my-preset 应当是一个包含 preset.json 的文件夹
vue create --preset ./my-preset my-project

# 或者，直接使用当前工作目录下的 json 文件：
vue create --preset my-preset.json my-project

# 插件开发

插件典型目录

.
├── README.md
├── generator.js  # generator (可选)
├── prompts.js    # prompt 文件 (可选)
├── index.js      # service 插件
└── package.json

  • service插件

    会在一个service实例被创建时自动加载，比如每次vue-cli-service命令在项目中被调用时

    // index.js
module.exports = (api, projectOptions) => {
  api.chainWebpack(webpackConfig => {
    // 通过 webpack-chain 修改 webpack 配置
  })

  api.configureWebpack(webpackConfig => {
    // 修改 webpack 配置
    // 或返回通过 webpack-merge 合并的配置对象
  })

  api.registerCommand('test', args => {
    // 注册 `vue-cli-service test`
  })
}

  • generator插件

    会在以下场景调用

    • 在一个项目的初始化创建过程中，如果cli插件作为项目创建preset的一部分被安装
    • 在项目创建好之后通过vue invoke独立调用时被安装
    // generator.js或generator/index.js
module.exports = (api, options, rootOptions) => {
  // 修改 `package.json` 里的字段
  api.extendPackage({
    scripts: {
      test: 'vue-cli-service test'
    }
  })

  // 复制并用 ejs 渲染 `./template` 内所有的文件
  api.render('./template')

  if (options.foo) {
    // 有条件地生成文件
  }
}

  • prompts插件对话

    在prompts.js文件里配置对话规则，如：

    module.exports = [
    {
        type: 'list',
        name: 'templateType',
        message: '请选择要创建的项目类型',
        choices: [
            {
                name: '默认',
                value: 'default',
                short: '默认'
            },
            {
                name: '手机',
                value: 'mobile',
                short: '手机'
            }
        ],
        default: 'default'
      },
      {
        name: 'publicPath',
        message: '请输入组件的上下文',
        default: '/template/'
      }
]

插件开发实战

  1. 创建项目

    $ mkdir vue-cli-plugin-test
$ cd vue-cli-plugin-test
$ npm init vue-cli-plugin-test -y

  2. 创建插件文件及文件夹

    .
├── README.md
├── generator
	├── template   # 自定义项目模板
	└──	generator.js  # generator
├── prompts.js    # prompt 文件
├── index.js      # service 插件
└── package.json

    // index.js
module.exports = {}

    // prompts.js
module.exports = [
    {
        type: 'list',
        name: 'templateType',
        message: '请选择要创建的项目类型',
        choices: [
            {
                name: '默认',
                value: 'default',
                short: '默认'
            },
            {
                name: '手机',
                value: 'mobile',
                short: '手机'
            }
        ],
        default: 'default'
      },
      {
        name: 'publicPath',
        message: '请输入组件的上下文',
        default: '/template/'
      }
]

    // generator.js
module.exports = (api, options, rootOptions) => {
	// 访问对话选项的值
    console.log(JSON.stringify(options))
    // 复制并用 ejs 渲染 `./template` 内所有的文件
    api.render('./template')
}

  3. 测试

    # 在vue-cli-plugin-test文件夹下执行：
$ npm link

# 创建一个vue项目并进入项目文件夹，然后执行：
$ npm link vue-cli-plugin-test
# 再去package.json的依赖里添加vue-cli-plugin-test
# 最后加载插件，执行：
$ vue invoke vue-cli-plugin-test

  4. 发布

    npm publish
